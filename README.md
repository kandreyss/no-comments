# no-comments

`no-comments` — это легкая и эффективная консольная утилита на языке C, предназначенная для автоматического удаления комментариев из исходного кода. Она поддерживает универсальный C-подобный синтаксис комментариев, что делает её совместимой с широким спектром языков программирования.

## Особенности

- **Удаление всех типов комментариев**:
  - Однострочные: `// ...`
  - Многострочные: `/* ... */`
- **Сохранение Doxygen-документации**: Утилита достаточно умна, чтобы не удалять комментарии, предназначенные для генерации документации (`/**`, `/*!`, `///`, `//!`).
- **Интеллектуальная очистка**: 
  - Удаляет лишние пробелы в конце строк (trailing whitespace), оставшиеся после удаления комментариев.
  - Предотвращает появление лишних пустых строк, сохраняя структуру кода чистой.
- **Безопасная работа с файлами**: При редактировании файла "на месте" (in-place) программа использует временный файл, чтобы гарантировать сохранность данных в случае сбоя.

## Поддерживаемые языки

Программа протестирована и корректно работает с языками, использующими C-стиль комментариев:
- **C / C++**
- **Java**
- **JavaScript / TypeScript**
- **Swift**
- **PHP**
- **Scala / Go**
- и другие.

## Сборка

Для компиляции программы вам потребуется компилятор `gcc` и утилита `make`.

```bash
make
```

После сборки будет создан исполняемый файл `no-comments`.

## Использование

### 1. Перезапись существующего файла (in-place):
Полезно для быстрой очистки файла.
```bash
./no-comments path/to/source.c
```

### 2. Сохранение результата в новый файл:
Используйте этот метод, если хотите сохранить оригинал.
```bash
./no-comments input.java output.java
```

### 3. Удаление:
Если вы захотите удалить программу из системы, выполните:
```bash
sudo rm /usr/local/bin/no-comments

## Структура репозитория

- `no-comments.c` — основной исходный код утилиты.
- `Makefile` — скрипт для сборки.
- Папки `cpp/`, `java/`, `swift/`, `php/` и др. — содержат примеры кода на соответствующих языках для тестирования работы парсера.
